# Guidelines — i18n (AIC)

## Locales

Arquivos em `frontend/public/i18n/`:

- `en.json` — inglês (fallback)
- `es.json` — espanhol
- `pt-BR.json` — português do Brasil

## Regras

- Toda chave nova deve existir nos **três** arquivos
- Usar namespaces em UPPER_SNAKE (`EXAMPLE.TITLE`, `COMMON.SAVE`)
- Preferir `| translate` no template; `TranslateService` no TS quando necessário
- Seletor: `app-language-switcher` + `I18nService`
- Storage: `aic.lang`

## Tools MCP

- `check_i18n_parity` — encontra chaves faltantes
- `get_i18n_key_template` — gera bloco JSON para um prefixo

## Checklist ao adicionar UI

1. Definir prefixo (ex. `MEMBERS`)
2. `get_i18n_key_template`
3. Preencher textos nos 3 idiomas
4. `check_i18n_parity`
